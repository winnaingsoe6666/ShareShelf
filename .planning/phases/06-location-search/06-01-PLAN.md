<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → ItemController.createItem | Untrusted lat/lng values cross this boundary via CreateItemRequest JSON body |
| Database → Item entity | PostGIS validates geometry type and SRID at the column level |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | CreateItemRequest.latitude/longitude | mitigate | `@field:Min(-90) @field:Max(90)` on latitude; `@field:Min(-180) @field:Max(180)` on longitude. Bean Validation rejects out-of-range coordinates with 400 BAD_REQUEST via existing MethodArgumentNotValidException handler. |
| T-06-02 | Tampering | Item.location column | mitigate | PostGIS `geometry(Point, 4326)` type enforces valid SRID 4326 geometry at the database level. Invalid geometries are rejected by PostgreSQL before reaching application code. |
| T-06-03 | Denial of Service | Missing GiST index | mitigate | V12 Flyway migration creates `idx_items_location` GiST index in the same transaction as the column addition. Without this index, spatial queries would sequential-scan the entire items table. Verified by SpatialIndexTest in Plan 06-02. |
| T-06-SC | Tampering | npm/pip/cargo installs | mitigate | slopcheck unavailable at research time. All packages tagged [ASSUMED]. Blocking human checkpoint gate (Task 0) verifies hibernate-spatial on Maven Central before Gradle dependency is added. |
</threat_model>
