<objective>
Replace the local filesystem storage backend with Cloudflare R2 (S3-compatible) for item photo uploads.

Purpose: Railway containers have ephemeral disk — local file storage means images are lost on every deploy. R2 provides persistent, CDN-backed object storage with no egress fees. The existing API contract (POST /items/{id}/images, DELETE /items/{id}/images) and the imageUrls JSONB column on items remain unchanged — only the storage layer swaps.

Output: AWS S3 SDK dependency, R2Config bean, refactored FileStorageService using S3Client, updated application configs, removed local file serving.
</objective>

       - Rewrite `store(file: MultipartFile, subPath: String): String`:

         - Generate UUID filename (keep existing logic)
         - Build S3 key: `"$subPath/$uuidFilename.$extension"`

       - Rewrite `delete(urlPath: String)`:
         - Extract S3 key from the URL by stripping the publicUrl prefix
         - Build `DeleteObjectRequest` with bucket and key
         - Call `s3Client.deleteObject(request)` in try-catch (best-effort, log warning on failure)
       - Remove all `java.nio.file.*` imports and local filesystem code

       - R2 public URLs are served from the R2 CDN domain, not from the backend
