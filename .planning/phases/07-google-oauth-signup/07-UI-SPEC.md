## UI Element Specifications

### 1. Google Sign-In Button

**Location:** Login page and register page, above the email/password form, inside the white card below the heading subtitle.

**Structure:**
```
[Google "G" logo SVG]  Sign in with Google
```

**Styling (Tailwind classes):**
- Container: `w-full flex items-center justify-center gap-3 px-4 py-2.5 rounded-lg border border-gray-300 bg-white text-gray-700 text-sm font-medium shadow-sm hover:bg-gray-50 hover:shadow-md transition-all duration-200 cursor-pointer`
- Google "G" logo: 20x20px inline SVG (official Google multi-color G)
- Text: 14px, weight 500, color #3C4043

**Interaction:**
- Hover: background lightens to #F8F9FA, shadow increases
- Click: navigates browser to `/api/oauth2/authorization/google` (Spring Security's default OAuth2 authorization endpoint)
- Loading: no client-side loading state needed (full-page redirect)

**Accessibility:**
- `aria-label="Sign in with Google"` on the link/button
- Focus ring: `focus:ring-2 focus:ring-purple-500 focus:ring-offset-2` (matches existing Button component)
- Keyboard: Enter/Space triggers navigation

### 2. Auth Divider

**Structure:**

### 5. Auth Callback Page

**Route:** `frontend/src/app/[locale]/auth/callback/page.tsx`

**States:**

#### Loading State (default)
```
Full-screen centered
┌─────────────────────────────────┐
│                                 │
│     [Loader2 spinning icon]     │
│                                 │
│       Signing you in...         │
│                                 │
│  Please wait while we complete  │
│   your authentication.          │
│                                 │
└─────────────────────────────────┘
```

**Styling:**
- Background: `min-h-screen bg-purple-50` (matches login/register)
- Card: `bg-white rounded-2xl shadow-xl p-8 text-center max-w-md`
- Icon: `<Loader2 className="mx-auto h-10 w-10 text-purple-400 animate-spin" />`
- Heading: `font-heading text-2xl font-bold text-purple-900`
- Body: `text-sm text-stone-600 mt-2`

#### Error State
```
Full-screen centered
┌─────────────────────────────────┐
│                                 │
│      [AlertCircle icon]         │
│                                 │
│     Authentication failed       │
│                                 │
│  Something went wrong during    │
│  sign-in. Please try again.     │
│                                 │
│      [Back to Sign In]          │
│                                 │
└─────────────────────────────────┘
```

**Styling:**
- Icon: `<AlertCircle className="mx-auto h-10 w-10 text-red-400" />`
- Heading: `font-heading text-2xl font-bold text-purple-900`
- Body: `text-sm text-stone-600 mt-2`
- Button: existing `Button` component, `variant="outline"`, linking to `/login`

**Behavior:**
- On mount: read `token` and `refreshToken` from URL search params
- If both present: call `saveAuth({ token, refreshToken, ... })`, then `router.push(returnUrl || "/items")`
- If missing/invalid: show error state
- Return URL: read from `returnUrl` search param (preserved from pre-redirect)

## Interaction Flows

### OAuth Login Flow (Frontend)

1. User clicks "Sign in with Google" on login or register page
2. Browser navigates to `/api/oauth2/authorization/google` (full-page redirect)
3. Spring Security redirects to Google consent screen
4. User authenticates with Google
5. Google redirects to backend `/oauth2/callback/google`
6. Backend exchanges code, creates/finds user, generates JWT
7. Backend redirects to `/auth/callback?token=xxx&refreshToken=yyy&returnUrl=...`
8. Callback page reads params, calls `saveAuth()`, redirects to return URL or `/items`

### Error Handling (Frontend)

- **Missing token params:** Show error state with "Back to Sign In" link
- **Invalid token:** Show error state (same as missing)
- **Network error during saveAuth:** Show error state
- **User cancels Google consent:** Google redirects back with `error` param; backend should handle and redirect to `/login?error=google_cancelled` (Claude's discretion)
