Modify handleSubmit flow:
After successfully creating the item via POST /items and getting the item ID, upload each file sequentially:
```typescript
// After: const res = await api.post("/items", {...});
// Before: router.push(`/items/${res.data.data.id}`);
if (res.data.success) {
  const itemId = res.data.data.id;
  if (selectedFiles.length > 0) {
    setUploadingImages(true);
    for (const file of selectedFiles) {
      const formData = new FormData();
      formData.append("file", file);
      await api.post(`/items/${itemId}/images`, formData);
    }
    setUploadingImages(false);
  }
  router.push(`/items/${itemId}`);
}
```

IMPORTANT: When calling `api.post()` with FormData, the Axios instance's default `Content-Type: application/json` header will be automatically overridden by the browser to `multipart/form-data` with boundary because FormData is the body. This is standard Axios behavior — no manual header manipulation needed.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client browser to Backend API | Image files uploaded via multipart/form-data — untrusted binary content crosses this boundary |
| Client to Backend (DELETE param) | URL query parameter — path traversal risk if backend doesn't normalize |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | POST /items/{id}/images | mitigate | Backend FileStorageService validates file extensions (jpg, png, gif, webp); frontend adds accept attribute on file input for defense-in-depth. Malicious file uploads rejected by backend. |
| T-02-02 | Tampering | DELETE /items/{id}/images?url= | mitigate | Backend normalizes path with .normalize() and validates against upload directory; frontend uses encodeURIComponent on the URL param. Path traversal attacks are blocked at backend level. |
| T-02-03 | Information Disclosure | Image URLs in API responses | accept | Image URLs are already public (served under /uploads/). No PII in image filenames. Low-value target. |
| T-02-04 | Denial of Service | Image upload endpoint | mitigate | Frontend limits to 5 files per item (maxFiles). Backend should enforce file size limits (existing concern outside this plan's scope). |
| T-02-SC | Tampering | npm install | mitigate | slopcheck + blocking human checkpoint for any [ASSUMED]/[SUS] packages. lucide-react is already in package.json — no new npm installs needed. |
